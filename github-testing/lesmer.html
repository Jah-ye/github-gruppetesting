Brukerveiledning i GitHub

Start med å opprette et repository på GitHub som gruppen skal jobbe i. Deretter lager dere issues for å dele opp oppgaver og få oversikt over hva som skal gjøres. Hver person jobber i sin egen branch for å unngå konflikter. Når du har gjort endringer, committer du koden med en tydelig melding og pusher den til GitHub. Etterpå oppretter du en pull request slik at de andre kan se gjennom arbeidet ditt. Til slutt blir endringene godkjent og merget inn i main-branchen. Dette er en enkel arbeidsflyt som hjelper dere å samarbeide strukturert og effektivt.
